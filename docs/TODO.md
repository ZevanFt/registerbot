- 增加统一步骤字典：`browser` 与 `http` 两套步骤定义（key/label/order）。

2. 后端统计对齐
- 返回每一步成功/总数统计时，按 `step key` 维度输出。
- 保证前端可直接按 key 匹配，不依赖中文文案。

3. UI 行为
- 切换注册类型时，监控卡片标题、副标题、步骤条、总成功率说明同步切换。
- 默认展示当前类型；历史类型数据分开，不混算。

4. 验收标准
- `browser` 下显示浏览器链路步骤。
- `http` 下显示纯 HTTP 链路步骤。
- 两种类型切换后，步骤数、步骤名、成功率均正确更新。
- 侧边“注册服务器”卡片与实际执行 Pipeline 步骤一致。
